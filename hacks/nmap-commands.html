What is Nmap?

Nmap (Network Mapper) is a free, open-source tool for network discovery and security auditing. It's used to discover hosts, services, operating systems, and vulnerabilities on a network.

Basic Syntax

nmap [Scan Type] [Options] {target}

Target Specification

Single IP

nmap 192.168.1.1

Hostname

nmap scanme.nmap.org

IP Range

nmap 192.168.1.1-254

Subnet/CIDR Notation

nmap 192.168.1.0/24

Multiple Targets

nmap 192.168.1.1 192.168.1.5 192.168.1.10

From File

nmap -iL targets.txt

Basic Scan Types

1. Ping Scan (Host Discovery)

Quickly check which hosts are online without port scanning

nmap -sn 192.168.1.0/24
Ping scan only, no port scan

2. Quick Scan

Scan the 100 most common ports

nmap -F 192.168.1.1
Fast scan of top 100 ports

3. Standard Scan

Scan the 1000 most common ports (default)

nmap 192.168.1.1
Default scan of top 1000 ports

4. All Ports Scan

Scan all 65,535 TCP ports

nmap -p- 192.168.1.1
Scan all ports (takes longer)

5. Specific Port(s)

nmap -p 80,443 192.168.1.1
Scan specific ports (HTTP & HTTPS)
nmap -p 1-100 192.168.1.1
Scan port range 1-100

Advanced Scan Types

TCP SYN Scan (Stealth Scan)

Most popular and default scan type. Fast and stealthy.

sudo nmap -sS 192.168.1.1
TCP SYN scan (requires root/admin)

TCP Connect Scan

Completes full TCP connection. Slower but doesn't require privileges.

nmap -sT 192.168.1.1
Full TCP connection scan

UDP Scan

Scan UDP ports (slower than TCP scans)

sudo nmap -sU 192.168.1.1
UDP port scan

Combined TCP and UDP

sudo nmap -sS -sU -p T:80,443,U:53,161 192.168.1.1
Scan specific TCP and UDP ports

Service & Version Detection

Service Version Detection

nmap -sV 192.168.1.1
Detect service versions

Aggressive Version Detection

nmap -sV --version-intensity 5 192.168.1.1
Maximum version detection (slower)

Light Version Detection

nmap -sV --version-intensity 0 192.168.1.1
Lighter/faster version detection

OS Detection

Enable OS Detection

sudo nmap -O 192.168.1.1
Detect operating system

Aggressive OS Detection

sudo nmap -O --osscan-guess 192.168.1.1
Guess OS more aggressively

Timing & Performance

Timing templates range from 0 (slowest/stealthiest) to 5 (fastest/loudest)

Template Speed Use Case
-T0 (Paranoid) Very slow IDS evasion
-T1 (Sneaky) Slow IDS evasion
-T2 (Polite) Slower Less bandwidth usage
-T3 (Normal) Default Standard scanning
-T4 (Aggressive) Fast Fast networks
-T5 (Insane) Very fast Extremely fast networks
nmap -T4 192.168.1.1
Aggressive timing (recommended for most scans)

Output Options

Normal Output to File

nmap -oN output.txt 192.168.1.1

XML Output

nmap -oX output.xml 192.168.1.1

Grepable Output

nmap -oG output.txt 192.168.1.1

All Formats

nmap -oA output 192.168.1.1
Outputs to .nmap, .xml, and .gnmap files

NSE Scripts (Nmap Scripting Engine)

Default Scripts

nmap -sC 192.168.1.1
Run default NSE scripts

Specific Script

nmap --script=http-title 192.168.1.1

Multiple Scripts

nmap --script=http-enum,http-headers 192.168.1.1

Script Categories

nmap --script=vuln 192.168.1.1
Run all vulnerability detection scripts

Common Script Categories:

  • auth - Authentication bypass scripts
  • default - Default safe scripts
  • discovery - Network discovery scripts
  • exploit - Exploitation scripts
  • vuln - Vulnerability detection
  • safe - Scripts unlikely to crash services

List Available Scripts

nmap --script-help all
ls /usr/share/nmap/scripts/
View all NSE scripts on system

Comprehensive Scan Examples

Quick Network Overview

nmap -sn -T4 192.168.1.0/24
Fast host discovery on local network

Standard Penetration Test Scan

sudo nmap -sS -sV -O -T4 192.168.1.1
SYN scan with service/OS detection

Aggressive Comprehensive Scan

sudo nmap -A -T4 192.168.1.1
Enable OS detection, version detection, script scanning, and traceroute

Full Scan with All Features

sudo nmap -sS -sV -O -A -T4 -p- --script=vuln -oA full_scan 192.168.1.1
Complete scan: all ports, service versions, OS, scripts, vulnerability detection, save all output

Stealth Scan

sudo nmap -sS -T2 -f --data-length 200 -D RND:10 192.168.1.1
Slow, fragmented packets with decoys for stealth

Common Use Cases

Web Server Reconnaissance

nmap -p 80,443 --script=http-enum,http-headers,http-methods 192.168.1.1

Mail Server Scan

nmap -p 25,465,587,110,995,143,993 -sV 192.168.1.1

Database Server Scan

nmap -p 1433,3306,5432,27017 -sV 192.168.1.1
Scan common database ports (MSSQL, MySQL, PostgreSQL, MongoDB)

Vulnerability Scan

nmap -sV --script=vulscan,vuln 192.168.1.1

SMB/Windows Share Enumeration

nmap -p 445 --script=smb-enum-shares,smb-enum-users 192.168.1.1

Tips & Best Practices

Always start with a ping sweep (-sn) to find live hosts before doing detailed scans
Use -v or -vv for verbose output to see progress during long scans
Save your scan results with -oA for documentation and later analysis
Combine flags for efficiency: sudo nmap -sS -sV -O -A -T4 -oA output target
Use sudo or run as administrator for SYN scans and OS detection
Test on scanme.nmap.org - a server specifically set up for testing Nmap
